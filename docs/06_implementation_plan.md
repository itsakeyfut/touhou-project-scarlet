# 06 実装計画

## 概要

東方紅魔郷クローンの実装を19フェーズに分割した段階的開発計画。各フェーズは独立して動作確認できる単位で区切られている。

---

## フェーズ一覧

| フェーズ | タイトル | 主要成果物 |
|---|---|---|
| Phase 1  | プロジェクトセットアップ | ビルド可能なワークスペース |
| Phase 2  | コアシステム | プレイヤー移動・カメラ |
| Phase 3  | ショットシステム | プレイヤー弾の発射 |
| Phase 4  | 基本弾幕 | 敵弾パターン実装 |
| Phase 5  | 衝突・グレイズ | 判定システム |

## Phase 1: プロジェクトセットアップ

**目標**: ビルドが通り、黒い画面が表示される状態

### タスク
- [ ] Cargo.toml ワークスペース設定確認
- [ ] 各クレート (scarlet-core, scarlet-ui, scarlet-audio, scarlet-assets) のディレクトリ・Cargo.toml 作成
- [ ] `touhou-project-scarlet` バイナリクレートの `main.rs` 作成
- [ ] `AppState` enum の定義 (`states.rs`)
- [ ] `ScarletCorePlugin`, `ScarletUiPlugin`, `ScarletAudioPlugin`, `ScarletAssetsPlugin` のスタブ作成
- [ ] `DefaultPlugins` でウィンドウ設定（640×480、タイトル「東方紅魔郷」）
- [ ] `justfile` のコマンド確認 (`just run`, `just build`, `just check`)
- [ ] CI設定: `cargo check` + `cargo clippy -D warnings` が通る

**確認**: `just run` で 640×480 の黒いウィンドウが開く

## 優先度と依存関係

```
Phase 1 (基盤)
  └─ Phase 2 (移動)
       └─ Phase 3 (ショット)
            ├─ Phase 4 (弾幕)
            │    └─ Phase 5 (衝突)
            │         ├─ Phase 6 (アイテム)
            │         ├─ Phase 7 (ザコ敵)
            │         └─ Phase 8 (ボス)
            │              └─ Phase 9 (パワー/ボム)
            │                   └─ Phase 10 (Stage 1)
            │                        └─ Phase 11 (ゲームフロー)
            │                             ├─ Phase 12 (HUD)
            │                             └─ Phase 13 (Audio)
            │                                  ├─ Phase 14 (Stage 2-3)
            │                                  ├─ Phase 15 (Stage 4-5)
            │                                  └─ Phase 16 (Stage 6)
            │                                       ├─ Phase 17 (Extra)
            │                                       └─ Phase 18 (ポリッシュ)
            │                                            └─ Phase 19 (アート)
```

各フェーズの完了基準は `docs/roadmap/phase-XX.md` に記載する。
